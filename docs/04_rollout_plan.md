# 落地分期方案

## Phase 1: 可用版本

目标：

- 支持 TIMS / Base Risk / Concentration Risk 三类算法接口
- 建立事件总线与版本化快照
- 支持持仓变动触发账户级重算
- 支持重大市场事件触发 `underlying` 级矩阵重建

范围建议：

- 先覆盖股票、单股票期权、期货、期货期权
- 先实现单币种
- 先覆盖日盘和主要盘中风控场景

## Phase 2: 工业化版本

目标：

- 引入完整的曲线构建服务
- 支持公司行动双版本切换
- 支持热标的池预热
- 支持批量账户并发重算
- 支持砍仓触发快照输出

重点建设：

- 事件幂等
- 任务优先级队列
- 审计链路
- 实时监控

## Phase 3: 增强版本

目标：

- 新增 `Liquidity Risk`、`Dislocation Risk`
- 更细粒度的 Greeks 缓存
- 更完善的解释性输出
- 支持回放和压力测试

## 运维与监控建议

至少监控以下指标：

- 每类事件吞吐量
- 曲线重建时延
- 矩阵重建时延
- 账户重算时延
- 快照写入成功率
- 砍仓前置队列积压
- 事件重复消费率
- 缓存命中率

## 上线前必须验证的场景

- 标的大跳导致深虚值期权瞬间靠近执行价
- 波动率左翼突然抬升
- 利率与股息同时更新
- 拆股/特别分红生效日切换
- 大客户新增集中空头仓位
- 配置切版后新旧版本一致性验证

## 建议的验收标准

- 普通账户持仓变动后，秒级得到新快照
- 重大市场事件后，受影响热账户在目标 SLA 内完成重算
- 每份快照都可追溯到版本 bundle
- 任何公司行动生效事件都不会引用旧交割合约缓存
