# 事件触发矩阵

## 1. 事件分类原则

建议把事件分为三大类：

- `Global`：会影响大量或全部标的，如利率、配置、全市场压力事件
- `Underlying`：只影响某个或某组标的，如价格大跳、波动率曲面变化、公司行动
- `Account`：只影响某个账户，如新增持仓、平仓、行权指派、划转

## 2. 触发与处理矩阵

| 触发事件 | 典型来源 | 影响范围 | 主要处理动作 | 典型调用流 | 对哪些期权影响更大 | 优先级 |

## 3. 每类事件的落地处理建议

### 3.1 市场类事件

#### 标的价格大幅变动

处理建议：

1. 先更新 `spot` 版本，不要一上来全量重算全市场。
2. 由 `Impact Analyzer` 判断是否超过“只改价格层”阈值，还是超过“需要重建曲面”阈值。
3. 仅对持有该 `underlying` 或同产品组高相关 `underlying` 的账户发起重算。
4. 若是指数大跳或相关资产联动，按产品组批量下发。

影响更大的期权：

- 临近到期、近平值期权：`gamma` 高，价格变化最剧烈
- 原本深度虚值但在大跳后接近执行价的期权：风险会突然显性化
- 卖出翼侧期权的组合：在集中风险场景下非常危险

#### 波动率曲面显著变化

处理建议：

1. 重新拟合 `vol surface`
2. 按 `underlying + expiry bucket + wing bucket` 失效相关矩阵切片
3. Base Risk 和 Concentration Risk 先重算，因为它们更直接依赖二维场景
4. TIMS 可在价格场景不变时复用部分 spot grid，但建议仍绑定新曲面版本

影响更大的期权：

- 长天期、近平值期权：`vega` 大
- 深度虚值认沽：对左翼 `skew` 非常敏感
- 长天期深度虚值期权：对远端翼侧外推敏感

### 3.2 参考数据类事件

#### 利率曲线改动

处理建议：

1. 重建对应币种的 `rate curve`
2. 只刷新长天期或 rho 较大的矩阵切片，避免全量浪费
3. 对短天期近到期合约可采取较低优先级补算

影响更大的期权：

- 长天期期权
- 期货期权
- 深实值合约与高 carry 资产上的期权

#### 股息曲线改动

处理建议：

1. 对单股票和相关期权链整体升版
2. 对临近除息日的美式看涨优先重算
3. 对组合中的股票现货、期货和期权同时做一致性刷新

影响更大的期权：

- 单股票美式看涨
- 临近除息日的深实值看涨
- 长天期单股票期权

#### 公司行动

处理建议：

1. 公告阶段：预计算新合约规则与交割单位，提前预热新矩阵
2. 生效阶段：旧版本矩阵全部失效，按新交割规则重建
3. 强制重算所有相关账户，不能依赖旧快照平滑过渡

影响更大的期权：

- 拆股、并购、特别分红导致合约调整的单股票期权
- 长天期合约
- 临近除权除息且可提前行权的美式合约

### 3.3 配置类事件

#### 保证金算法参数变更

处理建议：

1. 新旧参数版本必须并存，避免一半账户算旧版、一半账户算新版
2. 若只改 offset 比例，可先做账户级重算
3. 若改场景节点、冲击大小、波动率参数，则要重建矩阵
4. 对外输出时显示当前快照绑定的是哪一版规则

影响更大的期权与组合：

- 高度依赖对冲互抵的跨产品组合
- 日历价差
- 同一 `underlying` 下大量期权与现货互抵组合

### 3.4 账户类事件

#### 持仓变化

处理建议：

1. 第一优先级是账户级重算，不要触发大范围矩阵刷新
2. 如果新持仓引入一个新的 `underlying`，先异步预热矩阵，再同步算账户
3. 对高风险账户可以走“快照秒级刷新 + 异步补全解释信息”

影响更大的期权：

- 新增短期权
- 新增集中度高的大额单名仓位
- 临近到期且保证金占用波动快的仓位

## 4. 事件优先级和执行顺序

建议优先级：

- `P0`
  - 公司行动生效
  - 重大市场波动
  - 持仓新增/平仓
  - 行权指派
  - 保证金规则版本切换
- `P1`
  - 波动率曲面常规刷新
  - 利率与股息曲线更新
  - 日切
- `P2`
  - 手工补算、报表类重跑

建议执行顺序：

1. 版本化接收事件
2. 识别受影响对象与依赖
3. 先重建曲线
4. 再重建价格/波动率矩阵
5. 再重算账户保证金
6. 再写快照与告警

## 5. 关键控制点

- 相同 `underlying` 的连续小波动事件要做合并，避免抖动。
- 配置事件必须串行生效，不能和旧版本并发混算。
- 公司行动生效日建议提前预热新矩阵，生效时快速切换。
- 账户重算要与“受影响标的集合”关联，而不是全量扫账户。
